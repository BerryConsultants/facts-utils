runFACTSR Documentation

Run FACTS MCMC Model from R

Usage

runFACTS(engine = c("contin","dichot","ME","TTE","ed_contin","ed_dichot","ed_tte"),
                    data.file = "patients.dat",
                    param.file = "nuk1_e.param",
                    mcmc.file.num = 0,
                    rng.seed,
                    exec.path = getwd())

Arguments

engine

Name of the FACTS engine to use. Can be one of "contin","dichot","ME","TTE" for the Core engines, or "ed_contin", "ed_dichot","ed_tte" for the Enrichment Design engines.

data.file

Name of the input data file. Default is 'patients.dat'. This file format should be exactly match the file format of the 'patients' file corresponding to the ones produced by FACTS for the design you set up in FACTS to specify the analysis model. (See the FACTS Execution Guides under the FACTS Help menu for details.)

param.file

Name of the FACTS '.param' file that specifies the model setup. Default is 'nuk1_e.param'.

mcmc.file.num

The MCMC output is written to a file named mcmcNNNNN.csv. This argument sets the NNNNN. So mcmc.file.num=1 will create an MCMC output file called mcmc00001.csv. Default value is 0.

rng.seed

Integer-valued random number generator seed. Will use the value from the '.param' file if unspecified.

exec.path

The path to the directory where the FACTS executable program is available. Default is the current working directory.

Details

The function runFACTS provides the funtionality to run a FACTS analysis with a user-provided dataset. In order to use this, a corresponding FACTS project must be set up to generate the '.param' file.

Value

status

The function returns a TRUE/FALSE to indicate a successful/failed execution. In case of errors, R error messages may be printed, and in case of a FACTS execution error, a file called 'error.txt' with the error description will be output.

mcmcNNNNN.csv

A file with the MCMC output. (See FACTS documentation for details.)
